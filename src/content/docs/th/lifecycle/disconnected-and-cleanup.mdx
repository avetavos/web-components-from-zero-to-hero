## เมื่อไหร่ที่ disconnectedCallback ทำงาน

`disconnectedCallback` จะถูกเรียกทุกครั้งที่ element ถูกลบออกจาก document ที่เชื่อมต่ออยู่ งานทำความสะอาดที่ควรทำที่นี่ได้แก่:

- ลบ event listener ที่เพิ่มไว้ใน `connectedCallback`
- ยกเลิก timer ของ `setInterval` หรือ `setTimeout`
- ปิด WebSocket connections
- Disconnect `MutationObserver`, `ResizeObserver` หรือ `IntersectionObserver`

ข้อควรระวังสำคัญ: `disconnectedCallback` จะ**ไม่**ถูกเรียกเมื่อ page ทั้งหมดถูก unload (เช่น ผู้ใช้เปิดหน้าใหม่หรือปิด tab) มันจะถูก trigger เฉพาะจากการลบ DOM อย่างชัดเจน เช่น `removeChild`, `replaceWith` หรือการกำหนดค่าใหม่ให้ `innerHTML`

## รูปแบบ Symmetry

คิดว่า `connectedCallback` และ `disconnectedCallback` เป็นเหมือนวงเล็บที่จับคู่กัน ทรัพยากรที่คุณได้มาเมื่อ element เข้าสู่ DOM คุณก็ต้องคืนเมื่อมันออกไป:

```js
class TimerDemo extends HTMLElement {
  connectedCallback() {
    // เริ่มทำงาน
    this._intervalId = setInterval(this._tick.bind(this), 1000);
  }

  disconnectedCallback() {
    // หยุดทำงาน — สมมาตรกับ connectedCallback
    clearInterval(this._intervalId);
  }

  _tick() {
    // ถูกเรียกทุก 1 วินาทีขณะที่ connected
  }
}
```

การเก็บ handle (`this._intervalId`) ไว้บน element instance คือวิธีมาตรฐานเพื่อให้ `disconnectedCallback` เข้าถึงได้

## รูปแบบ Timer ที่สมบูรณ์

```js
class TimerDemo extends HTMLElement {
  connectedCallback() {
    this._seconds = 0;
    this._render();
    this._intervalId = setInterval(this._tick.bind(this), 1000);
  }

  disconnectedCallback() {
    clearInterval(this._intervalId);
  }

  _tick() {
    this._seconds += 1;
    this._render();
  }

  _render() {
    this.textContent = 'Running: ' + this._seconds + 's';
  }
}

customElements.define('timer-demo', TimerDemo);
```

export const cleanupJs = `class TimerDemo extends HTMLElement {
  connectedCallback() {
    var self = this;
    self.style.display = 'block';
    self.style.padding = '.5rem 1rem';
    self.style.background = '#fef3c7';
    self.style.borderRadius = '6px';
    self.style.marginTop = '.5rem';
    self._seconds = 0;
    self.textContent = 'Running: 0s';
    self._intervalId = setInterval(function() {
      self._seconds = self._seconds + 1;
      self.textContent = 'Running: ' + self._seconds + 's';
    }, 1000);
    var log = document.getElementById('log');
    if (log) { var p = document.createElement('p'); p.style.margin = '0'; p.textContent = '> connectedCallback: timer started'; log.appendChild(p); }
  }
  disconnectedCallback() {
    clearInterval(this._intervalId);
    var log = document.getElementById('log');
    if (log) { var p = document.createElement('p'); p.style.margin = '0'; p.textContent = '> disconnectedCallback: timer stopped'; log.appendChild(p); }
  }
}
customElements.define('timer-demo', TimerDemo);`;

<LivePreview client:visible html={cleanupHtml} js={cleanupJs} />

คลิก **Remove timer** เพื่อลบ element ออกจาก DOM — บรรทัด log ยืนยันว่า `disconnectedCallback` ทำงานและ timer หยุดแล้ว คลิก **Re-add timer** เพื่อแทรก element ใหม่อีกครั้ง counter จะ reset และ log จะแสดง `connectedCallback` ทำงานอีกครั้ง

<Tip title="เก็บ cleanup handle ไว้บน instance">
กำหนด interval ID, timeout ID หรือ reference ของ event handler ไว้เป็น property บน element instance (เช่น `this._intervalId`) ภายใน `connectedCallback` แล้ว `disconnectedCallback` จะสามารถหาและยกเลิกมันได้เสมอ
</Tip>
